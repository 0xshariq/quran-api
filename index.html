📖 Quran API Documentation

Explore the Holy Quran through our RESTful API. Built with Node.js and Express.

⚠️ API Key Required: All endpoints require authentication via API key.

Base URL

https://quran-api-ny11.onrender.com/api/v2

🔑 Authentication

Get Your API Key

1. Register or login at our authentication service:

POST https://user-authentication-api-jqfm.onrender.com/api/v2/register
POST https://user-authentication-api-jqfm.onrender.com/api/v2/login

2. You'll receive your API key via email after successful registration/login.

Using the API Key

Include the API key in the header :

x-api-key : YOUR_API_KEY

Example Request Header:


  "headers": {
    "x-api-key: "eyiOiJIUzI1NiIsInR5cCI6IkpXVCJ9"
  }

Incude the API key in the query :

?key : YOUR_API_KEY

Example Request Query:

        
          ?key=eyiOiJIUzI1NiIsInR5cCI6IkpXVCJ9

📚 Endpoints

1. Get All Surahs

Endpoint: GET /surahs

Headers:

x-api-key : YOUR_API_KEY

Query:

?key : YOUR_API_KEY

Example Request:

GET /surahs

2. Get Surah by Number

Endpoint: GET /surah/:surahNumber

Headers:

x-api-key : YOUR_API_KEY

Query:

?key : YOUR_API_KEY

Example Request:

GET /surah/1

Sample Response:

{
  "number": 1,
  "name": "الفاتحة",
  "englishName": "Al-Fatiha",
  "englishNameTranslation": "The Opening",

      "page": 1
    },
    ...

3. Get Ayah by Surah and Verse Number

Endpoint: GET /{surahNumber}:{verseNumber}

Headers:

x-api-key : YOUR_API_KEY

Query:

?key : YOUR_API_KEY

Example Request:

GET /1:1

Sample Response:

{
  "number": 1,
  "text": "بِسْمِ اللَّهِ الرَّحْمَٰنِ الرَّحِيمِ",
  "numberInSurah": 1,
  "juz": 1,
  "page": 1,
  "surah": {
    "number": 1,
    "name": "الفاتحة",
    "englishName": "Al-Fatiha"
  }
}

🚨 Error Responses

Invalid/Missing API Key

{
  "status": 401,
  "error": "Unauthorized",
  "message": "Valid API key required"
}

🔒 Security Best Practices

  • Never commit API keys to version control
  • Store keys in environment variables
  • Rotate keys periodically
  • Revoke compromised keys immediately
